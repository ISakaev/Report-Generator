Пояснительная записка.

Создание текстовых отчетов.
В классе Main указываем пути для файла с настройками, файла с входными данными а так же путь сохранения файла отчета.
Класс Main начинает работу с парсинга файлов: 

TSVParser парсит файл с данными и создает список с объектами каждой записи(Record).

XMLParser парсит файл с настройками и создает объект класса Settings, в который входит объект класса Page с размерами таблицы,
и список объектов Column с данними каждого столбца.

Следом начинается создание отчета:

Создается объект класса Report  у которого запускается метод createReport() и saveReport().
В методе createReport() класса Report создаеся объект класса ReportPage, который формирует листы для отчета.

Листы формируются путем добаления заголовка (объект класса HeadOfTable), разделителей (объект класса Separator) 
и списка строк записи (формируются в классе Field методом createField).
Так же отслуживается высота страницы отчета и по ее завершении добавляется разделитель "~".

Разделение данных поля ФИО происходит по след принцыпу:
Сначала проверяется есть ли пробел в строке размером с колонку + 1 символ(Пример "Ким Чен Ир"), 
далее если нет символа пробела, проверяется симлов дефис и разделени происходит по нему.
Если никаких символов нет, то строка обрезается по размеру колонки и остаток переносится на след строку(Пример "Павел Константинов").

пункт п) требований заменен : Границей слова является провел либо дефис. 
(Приоритет отдается пробелу в случае наличия обоих символов в строке.)
